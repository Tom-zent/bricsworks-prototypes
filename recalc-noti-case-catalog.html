재계산 미실행 노티 — 화면 상태 카탈로그

화면 톤(색상·버튼·단계 스텝퍼)은 브릭스웍스 실제 화면 기준으로 통일했고, 알림 카드는 기존 "선행 단계 데이터 변경" 알림, 제출 팝업은 기존 전자파일 제출 확인 팝업과 같은 스타일을 재사용했습니다. 노출 지점은 검증하기 + 전자파일 제출 2곳이며, 안내는 수정이 발견된 가장 앞선 단계를 기준으로 합니다 (앞 단계 수정 시 이후 단계 순차 재계산 필요).

예비검토 단계에서 값이 수정된 후 재계산이 진행되지 않았습니다. 예비검토 단계부터 이후 단계(사전작업 → 본검토 → 작성)를 순서대로 재계산한 뒤 다시 검증해 주세요. 그래도 진행하시려면 아래 버튼을 눌러주세요.

기존 알림은 단계 진입 시 자동으로 뜨는 안내형(액션 버튼 없음)이고, 이번 건은 검증하기·전자파일 제출 클릭이라는 명확한 시점에만 뜨며 진행 액션이 추가된다는 점이 차이입니다. 색상·아이콘·카드 구조는 동일하게 맞췄습니다.
