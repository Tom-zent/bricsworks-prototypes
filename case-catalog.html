환급계좌 키인 유효성 검증 — 상태별 전체 케이스

세부기획서 §5-2(검증 규칙)·§5-3(상태별 케이스)·§5-5(기존 입력분 처리)에 정의된 모든 상태를 케이스별로 정리했다. 각 케이스는 발생 조건 → 시스템 동작 → 노출 문구 → 해당 상태의 UI 순으로 구성된다. 마지막 절에서 실제로 조작해볼 수 있다.
원칙: ⑯ 국세환급금 계좌신고 행은 직접 수정 불가(읽기 전용)이며 값만 표기된다. 예금주 확인은 하지 않으며 계좌 실재 여부만 확인한다. API 장애 케이스는 통제 불가능한 변수라 정식 스펙에서 제외했다.
목차 버튼 위치·편집모드 CASE 0. 기본 상태 (기존 입력분) CASE 1. 필수값 미입력 CASE 2. 계좌 확인 불가 CASE 3. 성공 → 저장 CASE 4. 저장 후 ⑯행 반영 CASE 5. 미저장 변경사항 안내 인터랙티브 확인
[계좌 수정] 클릭→ (미저장 시 안내)→ ① 은행 선택→ ② 계좌번호 입력→ 계좌 유효성 체크→ 통과 시 [저장] 활성→ ⑯행 반영
버튼 위치 — [계좌 수정]은 작성 화면 하단 버튼 영역, [부속서류]와 [작성 결과] 사이에 고정 위치한다.
부속서류계좌 수정작성 결과도장 다운로드전체 화면

"수정하기" 편집모드와의 관계 — 상단 [수정하기]를 눌러 시트 편집 모드에 들어가면 상단은 [작성완료]만, 하단은 [부속서류]만 활성 상태를 유지하고 나머지는 비활성화된다. 이때도 [계좌 수정]은 예외적으로 계속 활성 상태다 — 계좌 수정은 시트 편집과 별개의 독립 동작이기 때문이다.
