환급계좌 키인 유효성 검증 — 상태별 전체 케이스

세부기획서 §5-2(검증 규칙)·§5-3(상태별 케이스)·§5-5(기존 입력분 처리)에 정의된 모든 상태를 케이스별로 정리했다. 각 케이스는 발생 조건 → 시스템 동작 → 노출 문구 → 해당 상태의 UI 순으로 구성된다. 마지막 절에서 실제로 조작해볼 수 있다.
원칙: ⑯ 국세환급금 계좌신고 행은 직접 수정 불가(읽기 전용)이며 값만 표기된다. 예금주 확인은 하지 않으며 계좌 실재 여부만 확인한다. 검증 상태(검증완료/미검증)는 팝업 상단 배지로 노출되며, 한 번 검증완료되면 이후 재검증 없이 계속 유지된다.
목차 버튼 위치 CASE 0. 기본 상태 — 레거시(직접입력) 건 CASE 1. 필수값 미입력 CASE 2. 계좌 확인 불가 CASE 3. 성공 → 저장 CASE 4. 저장 후 ⑯행 반영 CASE 5. 미저장 변경사항 안내 인터랙티브 확인
[계좌 수정] 클릭→ (미저장 시 안내)→ ① 은행 선택→ ② 계좌번호 입력→ 계좌 유효성 체크→ 통과 시 [저장] 활성→ ⑯행 반영
버튼 위치 — [계좌 수정]은 작성 화면 하단 버튼 영역, [부속서류]와 [작성 결과] 사이에 고정 위치하며, 나머지 하단 버튼들과 동일한 스타일(항상 활성)로 표시된다. 별도 토글·비활성 조건 없이 클릭 시 바로 모달이 열린다.
부속서류계좌 수정작성 결과도장 다운로드전체 화면

기본 상태 — 레거시(직접입력) 건

§5-5 기존 입력분 처리
발생 조건기능 배포 시점에 이미 자유 텍스트로 계좌가 입력돼 있던 딜. 은행 코드로 인식되지 않는 값(예: "신한3")
⑯행 표기기존 값이 그대로 표기된다(별도 표시 없음)
모달 진입 시배지는 [미검증]. 은행 선택 자리에 "직접입력"이 표시되고, 그 우측에 원본 은행명을 보여주는 읽기전용 참고박스가 뜬다. 계좌번호는 원본 숫자가 프리필되어 있다
전환같은 자리에서 실제 은행을 선택하면 참고박스가 사라지고 일반 은행 선택 UI로 바뀐다. 한 번 전환되면 "직접입력" 상태로 다시 돌아갈 수 없다
검증완료 후유효성 체크 통과 + 저장하면 [검증완료]로 영구 전환. 이후 다시 열어도 재검증 없이 저장된 은행·계좌번호가 그대로 나오고 배지도 계속 [검증완료]
