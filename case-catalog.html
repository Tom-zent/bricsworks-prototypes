환급계좌 키인 유효성 검증 — 상태별 전체 케이스

세부기획서 §5-2(검증 규칙)·§5-3(상태별 케이스)·§5-5(기존 입력분 처리)에 정의된 모든 상태를 케이스별로 정리했다. 각 케이스는 발생 조건 → 시스템 동작 → 노출 문구 → 해당 상태의 UI 순으로 구성된다. 마지막 절에서 실제로 조작해볼 수 있다.
원칙: ⑯ 국세환급금 계좌신고 행은 직접 수정 불가(읽기 전용)이며 값만 표기된다. 예금주 확인은 하지 않으며 계좌 실재 여부만 확인한다. 검증 상태(검증완료/미검증)는 팝업 상단 배지로 노출되며, 은행·계좌번호가 바뀌면(입력이든 삭제든) 실시간으로 체크·저장 버튼 활성 여부가 재평가된다.
목차 기본 화면 상태·버튼 위치 CASE 0. 기본 상태 — 레거시(직접입력) 건 CASE 1. 필수값 미입력 / 실시간 비활성 CASE 2. 계좌 확인 불가 CASE 3. 성공 → 저장 CASE 4. 저장 후 ⑯행 반영 · 재오픈 시 배지 유지 CASE 5. 미저장 변경사항 안내 인터랙티브 확인
[계좌 수정] 클릭→ (미저장 시 안내)→ ① 은행 선택→ ② 계좌번호 입력→ 계좌 유효성 체크→ 통과 시 [저장] 활성→ ⑯행 반영
기본 화면 상태 — 작성 화면 진입 시 상단은 [작성완료]만 활성(재계산실행·초기화·출력/현황은 비활성), 하단은 [부속서류]·[계좌 수정]만 활성(작성결과·도장다운로드·전체화면은 비활성)이다. [계좌 수정]은 [부속서류]와 [작성 결과] 사이에 고정 위치한다.
